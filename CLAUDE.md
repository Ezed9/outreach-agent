# CLAUDE.md — Outreach Agent

## What This Does
AI-powered cold email outreach system. Reads leads from CSVs (generated by leads-agent), researches each business, decides whether to pitch **websites** or **AI automation** (whichever fits better), writes a fully personalised email, lets you batch-review, then sends via Gmail SMTP with automatic follow-ups at day 3, 7, 14, and 21.

## Running

```bash
# Install deps
pip install -r requirements.txt

# Draft + review + send emails for a leads CSV
python main.py leads_gyms_sydney.csv

# Check & send due follow-ups
python main.py --follow-ups

# See status of all outreach
python main.py --status

# Warmup mode (run daily for 4 weeks before cold outreach)
python warmup.py

# Verify emails in a CSV before sending
python main.py --verify leads_gyms_sydney.csv
```

## Environment Variables

Copy `.env.example` → `.env` and fill in:
- `GMAIL_ADDRESS` — Gmail account to send FROM (use a secondary domain, not your personal Gmail)
- `GMAIL_APP_PASSWORD` — Gmail App Password (not your regular password). Get at: Gmail → Settings → Security → 2-Step Verification → App Passwords
- `GMAIL_IMAP_HOST` — usually `imap.gmail.com`
- `GMAIL_SMTP_HOST` — usually `smtp.gmail.com`
- `YOUR_NAME` — Your first name (used in email signatures)
- `YOUR_CALENDLY` — Optional Calendly link for CTAs in follow-up emails
- `GEMINI_API_KEY` — Primary LLM for email drafting (free, 1500 req/day)
- `GROQ_API_KEY` — Fallback LLM
- `OPENROUTER_API_KEY` — Second fallback

## Architecture

**Main flow** (`main.py`):
1. Load leads from CSV → filter to those with email OR website
2. For each new lead: `drafter.py` researches the business → decides pitch → writes personalised email
3. `reviewer.py` shows batch for human review (Approve / Edit / Skip)
4. `sender.py` sends approved emails via Gmail SMTP, logs Message-ID
5. `tracker.py` saves state to `outreach_tracker.json`
6. `reply_checker.py` checks IMAP for replies when running `--follow-ups`
7. Follow-ups are drafted + sent for leads with no reply after 3/7/14/21 days

**LLM Backend** (`agent.py`): Same chain as leads-agent — Gemini first, falls back to Groq → OpenRouter → Ollama. No rate limits when Ollama is running locally.

**Email Personalisation** (2-step):
1. Scrape business website → extract: what they do, services, contact style, gaps (no booking, no website, manual processes)
2. LLM decides pitch: website OR AI automation (whichever fits better)
3. LLM writes 75–100 word plain-text email:
   - Line 1: specific observation about their business
   - Lines 2–3: value prop tied to their situation
   - Line 4: soft CTA ("Worth a 15-min chat this week?")

**Tracker** (`outreach_tracker.json`): JSON file tracking every lead's outreach state. Schema:
```json
{
  "id": "hash(company_name+website)",
  "company_name": "Cutting Edge Barbershop",
  "to_email": "info@...",
  "website": "https://...",
  "pitch": "website",
  "status": "sent",
  "emails": {
    "1": {"subject": "...", "body": "...", "sent_at": "2026-03-19", "message_id": "<...>"},
    "2": {"subject": "...", "body": "...", "sent_at": null, "message_id": null},
    "3": null,
    "4": null,
    "5": null
  },
  "replied_at": null,
  "notes": ""
}
```

## Key Implementation Details

**Email sending rules (deliverability)**:
- Plain text only — no HTML, no images, no tracking pixels
- No links in email 1 — only raw URL in follow-ups if needed
- Max 30 emails/day (enforced in `sender.py`)
- 2-second delay between sends
- Message-ID stored for IMAP reply threading

**Follow-up timing**: Day 0 → Day 3 → Day 7 → Day 14 → Day 21
- Each follow-up is shorter, different angle
- After 5 emails no reply → status = "exhausted", stop

**Warmup** (`warmup.py`):
- Must run for 4 weeks before cold outreach
- Sends between your own accounts, rescues from spam, replies
- Schedule: week 1 = 5/day, week 2 = 15/day, week 3 = 30/day, week 4 = 50/day

**Email verification** (`tools/verifier.py`):
- DNS MX lookup + SMTP RCPT check (local, no paid API)
- Run before sending — bounces above 2% kill domain reputation

**Reply detection** (`tools/reply_checker.py`):
- Uses `imap_tools` with `In-Reply-To` header matching
- Searches INBOX for replies to sent Message-IDs
- Updates tracker status to "replied" when found

## Tools

- `tools/verifier.py` — MX + SMTP email verification
- `tools/reply_checker.py` — IMAP reply detection via imap_tools
- `tools/website_researcher.py` — httpx + html2text website scraper (adapted from leads-agent fetch_webpage)

## Critical Files

- `outreach/sender.py` — Gmail SMTP sender

## Before You Send a Single Email (Checklist)
1. [ ] Buy a secondary .com domain (Cloudflare, $10/yr) — never use personal Gmail
2. [ ] Set SPF, DKIM, DMARC DNS records for your domain
3. [ ] Run `warmup.py` daily for 4 weeks
4. [ ] Test at mail-tester.com — score must be 9+/10
5. [ ] Add domain to Google Postmaster Tools
6. [ ] Run `python main.py --verify <csv>` before each campaign

## Spam Words — Never Use
FREE, guaranteed, risk-free, act now, limited time, earn money, click here, winner, congratulations, unsubscribe, no obligation, 100%

## Adding New Lead Sources
Drop any `leads_*.csv` from the leads-agent directory into this folder and pass it as an argument. The system reads standard CSV columns: `company_name, score, source, description, website, email, linkedin, why_good_lead, url`
