## What This Does

AI-powered cold email outreach system. Reads leads from CSVs (generated by leads-agent), researches each business, decides whether to pitch **websites** (Max Web brand) or **AI automation** (Arkhe AI brand), writes a fully personalised email, auto-validates it, then sends via Gmail SMTP with automatic follow-ups at day 3, 7, 14, and 21. Fully automatic — no interactive approval step.

## Commands

```bash

# Core workflow
python main.py leads_gyms_sydney.csv          # Draft + send (automatic)

python test_send.py                            # Send one test email + log in tracker (for testing reply detection)
```

## Environment Variables

Copy `.env.example` → `.env` and fill in:
- `GMAIL_ADDRESS` / `GMAIL_APP_PASSWORD` — Gmail account to send FROM (use a secondary domain, not personal Gmail)
- `GMAIL_IMAP_HOST` — usually `imap.gmail.com`
- `GMAIL_SMTP_HOST` — `smtp.gmail.com` (Gmail) or `smtp-relay.brevo.com` (Brevo)
- `YOUR_NAME` — First name (used in email signatures)
- `AI_SITE_URL` — Arkhe AI website, included in AI automation follow-ups
- `WEBSITE_PORTFOLIO_URL` — Max Web portfolio, included in website follow-ups
- `BREVO_SMTP_USER` — Brevo SMTP login (e.g. `abc123@smtp-brevo.com`); required when `GMAIL_SMTP_HOST=smtp-relay.brevo.com`. `GMAIL_APP_PASSWORD` holds the Brevo SMTP key in this case.
- `GMAIL_IMAP_USER` / `GMAIL_IMAP_PASSWORD` — Gmail account used to **receive** replies (can differ from the sending address)
- `GEMINI_API_KEY` — Primary LLM (free, 1500 req/day)
- `GROQ_API_KEY` — Fallback LLM
- `OPENROUTER_API_KEY` — Second fallback

## Architecture

```
main.py                          Entry point, CLI arg parsing, orchestrates full pipeline
├── outreach/drafter.py          Research → pitch decision → LLM email drafting + validation
├── outreach/sender.py           Gmail SMTP sending with daily limits + delay
├── outreach/tracker.py          JSON state management (outreach_tracker.json)
├── agent.py                     LLM backend: Gemini → Groq → OpenRouter → Ollama fallback chain
├── models.py                    Dataclasses: Lead, EmailDraft, SentEmail, OutreachRecord
├── tools/website_researcher.py  httpx + html2text website scraper (tech detection, signals)
├── tools/reply_checker.py       IMAP reply detection via imap_tools + In-Reply-To header matching
├── tools/verifier.py            DNS MX lookup + SMTP RCPT email verification
├── warmup.py                    Email warmup script (4-week ramp: 5→15→30→50/day)
└── scripts/setup_check.py       Pre-flight SMTP/DNS/LLM key verification
```

**Main flow** (`main.py`):
1. Load leads from CSV → filter to those with email OR website
2. For each new lead: `drafter.py` researches the business → decides pitch → writes personalised email
3. Auto-approve: drafts with 0–1 warnings send automatically, 2+ warnings get skipped
4. `sender.py` sends emails via Gmail SMTP, logs Message-ID
5. `tracker.py` saves state to `outreach_tracker.json`
6. `reply_checker.py` checks IMAP for replies when running `--check-replies` or `--follow-ups`
7. Follow-ups are drafted + sent for leads with no reply after 3/7/14/21 days

**Status lifecycle** (in `models.py`): `pending` → `sent` → `replied` | `skipped` | `exhausted` (after 5 emails with no reply)

**LLM Backend** (`agent.py`): Gemini first (`gemini-2.0-flash`), falls back to Groq → OpenRouter → Ollama. No rate limits when Ollama is running locally.

## Key Implementation Details

**Email sending rules (deliverability)**:
- Plain text only — no HTML, no images, no tracking pixels
- No links in email 1 — only raw URL in follow-up 2 if needed
- Max 30 emails/day (enforced in `sender.py`)
- 2-second delay between sends
- Message-ID stored for IMAP reply threading

**Dual-brand pitching**:
- "website" pitch → **Max Web** brand, portfolio at `WEBSITE_PORTFOLIO_URL`
- "ai_automation" pitch → **Arkhe AI** brand, site at `AI_SITE_URL`
- Email 1: no URLs, brand name-dropped once casually, sign-off includes brand name
- Follow-up 2 only: includes raw service URL
- Follow-ups 3–5: no URLs, conversational
- Both brands send from same `GMAIL_ADDRESS`

**Follow-up sequence** (Day 0 → Day 3 → Day 7 → Day 14 → Day 21):
- Email 2 (day 3): NEW VALUE — industry stat/insight, never "just following up"
- Email 3 (day 7): PATTERN INTERRUPT — ultra-short (20–40 words), one direct question
- Email 4 (day 14): SOCIAL PROOF — brief case study for similar business type
- Email 5 (day 21): BREAKUP — warm close, zero pitch
- Angles defined in `FOLLOWUP_ANGLES` dict in `drafter.py`

**Email validation** (`_validate_email_draft()` in `drafter.py`):
- Runs after every LLM draft (initial + follow-ups), acts as the automated quality gate
- Checks: spam words, word count (50–125 for email 1, 20–70 for follow-ups), subject format (lowercase, max 6 words), CTA presence (email 1 must end with a question), no links in email 1
- On failure: re-prompts LLM once with specific fix instructions, keeps whichever version has fewer warnings
- Drafts with 0–1 warnings auto-send; 2+ warnings auto-skip (threshold: `MAX_WARNINGS_TO_SEND` in `main.py`)

**Email frameworks** (in `DRAFT_EMAIL_PROMPT`):
- LLM picks from AIDA, PAS, or BAB per lead — framework is never named in the email
- Subject lines: 2–4 words, all lowercase, no punctuation
- Banned openers: "I noticed...", "I came across..."
- Banned CTAs: "Worth a chat?", "Let me know", "Would love to connect"
- Spam words (FREE, guaranteed, risk-free, act now, etc.) trigger automatic LLM re-prompt

**Website research signals** (`tools/website_researcher.py`):
- WordPress/Wix/Squarespace/Shopify detection, copyright year freshness, hiring/expansion signals, review platform presence
- Signals flow: research → drafter (for LLM context) → summary table output

## Before You Send a Single Email (Checklist)

1. Buy a secondary .com domain — never use personal Gmail
2. Set SPF, DKIM, DMARC DNS records
3. Run `warmup.py` daily for 4 weeks
4. Test at mail-tester.com — score must be 9+/10
5. Add domain to Google Postmaster Tools
6. Run `python main.py --verify <csv>` before each campaign
